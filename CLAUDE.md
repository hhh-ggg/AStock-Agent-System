\# CLAUDE.md



你正在帮助我开发一个个人 A股智能投研、盘中买卖点提醒、持仓管理和复盘学习系统。



\## 绝对边界



1\. 只做 A股分析，不做加密货币。

2\. 不做自动交易。

3\. 不连接券商交易客户端。

4\. 不模拟点击券商软件。

5\. 不使用个人微信 hook、外挂、模拟登录。

6\. 所有买卖点、止损、止盈、仓位建议必须由程序规则计算。

7\. 大模型只负责解释、总结、公告新闻分析、复盘归因和提醒文案。

8\. 大模型不能直接决定买卖。

9\. 大模型不能直接修改交易规则。

10\. 核心风控参数不能自动放宽。

11\. 所有策略参数正式上线前必须人工确认。



\## 技术栈



\- Python 3.11+

\- FastAPI

\- PostgreSQL / TimescaleDB

\- Redis

\- Celery

\- APScheduler

\- SQLAlchemy

\- Alembic

\- Pydantic

\- Pandas / Polars

\- pytest

\- Docker Compose



\## 第一阶段 MVP



第一阶段只实现：



1\. FastAPI 项目骨架

2\. PostgreSQL 数据库模型

3\. mock 行情数据源

4\. 日线特征计算

5\. 市场环境评分

6\. 个股综合评分

7\. 风险过滤

8\. 持仓诊断

9\. 盘前候选股生成

10\. signal\_events 记录

11\. 每日复盘基础统计



暂时不要实现：



1\. 自动交易

2\. 真实券商接口

3\. Level-2 行情

4\. 复杂机器学习

5\. 个人微信 hook

6\. 自动修改策略参数

7\. 全市场秒级实时扫描



\## 开发要求



1\. 每次只实现一个小模块。

2\. 每个模块必须有 pytest。

3\. 所有策略参数必须配置化。

4\. 所有 API Key 必须从环境变量读取。

5\. 不得把密钥写入代码。

6\. 不得提交 .env。

7\. 所有 AI 输出必须通过 Pydantic schema 校验。

8\. 所有信号必须写入 signal\_events。

9\. 所有信号必须可以复盘。

10\. 数据源必须通过 provider adapter 抽象，不允许写死在业务逻辑里。



\## 工作方式



1\. 开始编码前，先给实现计划。

2\. 涉及数据库、策略、风控、AI 调用时，先解释设计再写代码。

3\. 每次修改后运行测试。

4\. 不要自动 git push。

5\. 不要删除文件，除非我明确要求。
